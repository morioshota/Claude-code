# KABU DEX を公開する（Vercelデプロイ＆AIアシスタント接続）

初めての方向けの手順書です。所要時間はだいたい20〜30分。
**手順1〜2だけでアプリの公開は完了**します。AIアシスタント（🤖AI下書き）も使いたい場合だけ手順3に進んでください。

## 全体像

- **Vercel** … GitHubのコードを自動でビルドして公開してくれるサービス。個人利用なら**無料**（Hobbyプラン）
- **Anthropic API** … AI下書き機能が裏で使うAI。こちらは**使った分だけ課金**（下書き1回あたり数円〜十数円程度。web検索は1,000回あたり$10 + 通常のトークン代）
- このリポジトリには設定ファイル（`vercel.json`）とAI用の中継サーバー（`api/ai-draft.js`）を用意済みなので、**コードを書く作業はありません**

---

## 手順1: Vercelアカウントを作る（5分）

1. https://vercel.com/signup を開く
2. **「Continue with GitHub」** を選ぶ（このリポジトリがあるGitHubアカウントでログイン）
3. 画面の指示に従って登録を終える（Hobbyプラン＝無料でOK）

## 手順2: このリポジトリをデプロイする（5分）

1. Vercelのダッシュボードで **「Add New...」→「Project」** をクリック
2. リポジトリ一覧から **このリポジトリ**（Claude-code）を選んで **「Import」**
   - 一覧に出ない場合は「Adjust GitHub App Permissions」からリポジトリへのアクセスを許可する
3. 設定画面が出るが、**何も変更しなくてよい**（Framework Preset が「Vite」と自動認識されていることだけ確認）
4. **「Deploy」** をクリックして1〜2分待つ
5. 🎉 が出たら完了。表示されたURL（`https://○○○.vercel.app`）がアプリの公開アドレス

> 以後、GitHubにpushするたびに自動で再デプロイされます。手動作業は不要です。

> **参考株価について**: デプロイ版では銘柄詳細に参考株価（Yahoo Financeの遅延データ）が自動表示されます。追加設定・APIキーは不要です。表示は事実のみ（価格・日時・出典）で、売買判断用ではありません。Yahoo Financeの非公式データを個人利用の範囲で使っています。

### ⚠ データについての大事な注意

- 図鑑のデータは**ブラウザごと（端末ごと）のlocalStorage**に保存されます。デプロイしてもデータが勝手に共有・同期されることはありません
- 手元のブラウザで使っていたデータを公開版に移すには、アプリの **「💾 バックアップ」→ 書き出し → 公開URLで読み込み** を使ってください
- URLを知っている人は誰でもアプリを開けます（他人のデータは見えません。各自のブラウザに保存されるだけです）

## 手順3: AIアシスタントをつなぐ（10分・任意）

「🤖AI調査アシスタント」ボタンを動かすための設定です。不要ならスキップしてOK。

### 3-1. AnthropicのAPIキーを取る

1. https://console.anthropic.com/ でアカウントを作る（claude.aiとは別アカウント体系）
2. 支払い方法を登録し、クレジットをチャージする（最少額でOK。**使いすぎ防止に、Settings → Limits で月の上限額を必ず設定**しておくのがおすすめ）
3. **Settings → API Keys → 「Create Key」** でキーを作る
4. 表示された `sk-ant-...` で始まる文字列をコピーする（**この画面でしか表示されません**。メモ帳などに一時保存）

### 3-2. VercelにAPIキーを登録する

1. Vercelのプロジェクト画面 → **Settings → Environment Variables**
2. 次のとおり入力して **「Save」**
   - **Key**: `ANTHROPIC_API_KEY`
   - **Value**: さっきコピーした `sk-ant-...`
   - **Environment**: すべて（Production / Preview / Development）のままでOK
3. 環境変数は**再デプロイ後に反映**されます: **Deployments** タブ → 一番上のデプロイの「⋯」→ **「Redeploy」**

### 3-3. 動作確認

公開URLでアプリを開き、銘柄の詳細 → 「🤖AI調査アシスタント」→ **「🔍 検索して下書きを生成する」**。
30秒ほどで下書きが表示されれば成功です。

> **エラーが出たら**
> - 「ANTHROPIC_API_KEY が設定されていません」→ 3-2 の環境変数名のスペルとRedeployを確認
> - 「credit balance is too low」等 → Anthropicコンソールでクレジット残高を確認
> - しばらく待ってもだめなら、Vercelの **Deployments → 該当デプロイ → Functions** でエラーログを見られます

### 仕組みとセキュリティ（読み物）

- APIキーは**Vercelのサーバー側だけ**に保存され、ブラウザには一切送られません。ブラウザは同じサイト内の `/api/ai-draft` を呼び、そこ（`api/ai-draft.js`）がAnthropicに中継します
- 中継サーバーは「許可したモデル」「トークン上限」「web検索ツールのみ」に固定してあるので、URLを知られても高額請求につながる操作はできない作りです。それでも心配なら、Anthropic側の**月額上限設定**が最後の砦になります

## ローカル開発でAIを使いたい場合（任意）

`npm run dev` のローカル環境ではプロキシが動かないため、デプロイ済みのプロキシを借ります。
プロジェクト直下に `.env.local` を作成:

```
VITE_ANTHROPIC_PROXY=https://あなたのURL.vercel.app/api/ai-draft
```

`.env.local` はgitに入れないでください（`.gitignore` で除外済み）。
