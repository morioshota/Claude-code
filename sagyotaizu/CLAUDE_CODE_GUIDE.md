# Claude Code 引き継ぎガイド

このツールを、以後 Claude Code（ターミナル / デスクトップから使うエージェント型のコーディングツール）で育てていくための立ち上げ手順。

---

## 0. 用意するもの
- Node.js（Claude Code の動作要件。導入時に案内される）
- このフォルダ一式（下記「同梱ファイル」）
- ブラウザ（動作確認用。Chrome/Edge 推奨）

## 同梱ファイル
```
作業帯図作成ツール.html   ← 本体（これを編集していく）
README.md                 ← 仕様書。何をする/しないツールか、データモデル
CODE_STRUCTURE.md         ← コードの地図。「どこを触るか」早見表
ROADMAP.md                ← 今後の改修候補（優先度つき）
CLAUDE_CODE_GUIDE.md      ← このファイル
```

---

## 1. フォルダを用意して置く
任意の作業フォルダ（例：`sagyotaizu/`）を作り、同梱ファイルを全部入れる。

```
sagyotaizu/
  作業帯図作成ツール.html
  README.md
  CODE_STRUCTURE.md
  ROADMAP.md
  CLAUDE_CODE_GUIDE.md
```

## 2. Claude Code を起動
そのフォルダをカレントにして Claude Code を起動する（アプリ版なら対象フォルダを開く）。
起動時、`README.md` と `CODE_STRUCTURE.md` を読ませておくと、以後のやり取りが速くなる。最初にこう伝えるとよい：

> このフォルダの `README.md` と `CODE_STRUCTURE.md` を読んで。`作業帯図作成ツール.html` は単一HTMLの作業帯図ツール。これから機能追加していくので、単一HTML・外部依存なし（ライブラリはCDN遅延ロードのみ）の制約は必ず守って。

## 3. 変更を依頼する（例）
`ROADMAP.md` の項目を指定すると、Claude Code が該当箇所を特定して編集する。例：

> `ROADMAP.md` の R1（縮尺バーと縮尺分母の直接入力）をやりたい。まず変更方針を出してから実装して。

Claude Code はファイル内を検索して必要な箇所（描画・UI・データモデル）を横断的に直せる。1機能=1まとまりで進めるのがおすすめ。

## 4. 動作確認
編集後は `作業帯図作成ツール.html` をブラウザで開いて確認するだけ（ビルド不要）。
確認する項目は `CODE_STRUCTURE.md` 末尾の「テスト観点」を使う。特に：
- PNG出力が崩れない
- 縮尺設定 → 寸法が実測値になる
- JSON保存 → 読込で復元される
- 印刷（A4/A3）が用紙にフィットする

問題があれば、その画面の様子を Claude Code に伝えれば直してくれる。

---

## 5. このツール特有の“守ってほしいこと”を伝える
Claude Code に最初に共有しておくと事故が減る要点：

1. **単一HTMLを崩さない**。CSS/JSは同一ファイル内。新規ライブラリは `ensurePdfJs()` と同じ**遅延ロード方式**で。
2. **`prompt()` / `alert()` を作図フロー中に使わない**（ブラウザにブロックされる）。入力は `askInput()` モーダルを使う。
3. **部材（オブジェクト型）を追加するときは、`CODE_STRUCTURE.md` の「触る7〜9箇所」を必ず全部**。`divzone` を grep すると見本が並ぶ。
4. **座標は世界px、実寸は `mPerPx` 換算**。記号サイズは `pxPerM()` 基準。縮尺連動を壊さない。
5. **変更後は必ずPNG出力とJSON往復を確認**。

---

## 6. バージョンの記録
ファイル冒頭のコメント（`作業帯図作成ツール v1.1 …`）に変更概要を1行足していく運用にすると、後から差分を追いやすい。大きめの変更のたびに `ROADMAP.md` のチェックを更新する。

---

## 補足：なぜチャットからClaude Codeへ移すのか
このツールは1500行規模に育ち、今後も伸びる。チャットで毎回ファイル全体を読み書きするより、Claude Codeで**差分だけを編集し、その場でブラウザ確認**する方が速く・安全で、単一HTML納品とも相性がよい。動く最新版（v1.1）を起点に引き継ぐことで、ゼロから説明し直す手間もない。
