# 土木3Dビルダー（civil-3d）

土木図面の **平面図（線形）** と **横断図** を組み合わせて、検討用の3Dモデルをブラウザ上で生成するプロトタイプです。
**試験掘りデータの確認・仮設（オープンカット）設計・構造物設置の当たり確認**に使うことを想定しています。

> このディレクトリは銘柄図鑑（KABU DEX）とは独立した新規プロジェクトです。同じリポジトリに同居しているだけで、依存関係もビルドも別です。

## できること（現状）

- **回廊サーフェス生成**: 平面線形に沿って横断面をロフトし、地表サーフェスを格子メッシュで生成
- **横断線 / 中心線の表示**: 各測点の横断ライン、線形の中心線
- **試験掘り（簡易柱状図）**: 位置・土層構成を色分けした縦の柱として3D表示。クリックで層名・層厚を表示
- **構造物の当たり確認**: ボックスカルバート等を半透明ボックスで配置（埋設深さ指定）
- **仮設オープンカットの表示**: 掘削範囲を半透明ボックスで表示。構造物との干渉を目視確認
- **縦倍率**: 高低差を1〜8倍に強調（土木の定番）
- **表示レイヤのON/OFF**、慣性つきカメラ操作（OrbitControls）
- **DWG / DXF 読み込み（図面ビュー）**: 実際の図面をブラウザで読み込み、レイヤ別に色分け表示。
  文字ラベルも表示（断面線 A-A'・B-B'、寸法、注記など）。LINE/ポリライン/円/円弧/楕円をレンダリング。
  DWGは LibreDWG(WASM) で直接読み込み、DXFは dxf-parser で処理。
- **3D起こし（平面図トレース→押し出し）**: 平面図の土留め/掘削の輪郭をクリックでトレースし、
  深さ（矢板長 L=3500 など）を入力して立坑(土留め/掘削)の3Dソリッドを生成。
  地表(GL)枠を表示し、**底面積・掘削体積**を自動算出。仮設・掘削の検討に直結。
- **試験掘り（柱状図）の3D配置**: 立坑モデル上で土層（名前・層厚）を入力し、GL面をクリックして
  試験掘りを配置。色分けした柱状図を3D表示し、**矢板/掘削の深さと土層の関係**（支持層に
  届くか・軟弱層の位置）を目視で検討できる。
- **構造物の配置と当たり確認**: MH・カルバート等を寸法（W×L×H・天端下がり）で配置し、
  半透明ボックスで立坑内の**納まり・干渉**を確認。試験掘り・掘削深さと合わせて総合検討できる。
- **横断図の半自動ジオリファレンス→現況地盤サーフェス**: 平面図で断面線の2端点、横断図で
  標高既知の2点（EL入力）を指定し、地盤ラインをトレースすると、正しい平面位置・標高スケールで
  断面を3D化。2本以上でロフトして**現況地盤サーフェス**を生成。図面の縦横スケールが未知でも成立。
- **統合ビュー（重ね合わせ）**: 現況地盤・立坑・試験掘り・構造物を**同一の実標高系**で重ねて表示。
  立坑/試験掘り/構造物のGL基準を地盤サーフェスの標高に載せ、掘削底のEL・掘削体積を提示。
  図面／立坑／地盤／統合をワンタッチで切替、要素ごとに表示ON/OFF。設置検討を1画面で完結。

`デモ現場を読み込む` を押すと、実DXFが無くてもサンプル現場（延長120mのカーブ道路＋現況地盤＋試験掘り3本＋ボックスカルバート）で全機能を確認できます。

## 使い方

```bash
cd civil-3d
npm install
npm run dev      # http://localhost:5180
npm run build    # dist/ に本番ビルド
```

## 設計上の考え方（重要 / 正直な線引き）

このツールの本質は「線の意味の解釈」です。平面図・横断図から3Dを起こす作業は、
**どの線が地盤線・構造物・法面・中心線か**を判断するところに難しさがあります。

そのため段階を分けています。

| 段階 | 入力 | 実現性 | 本ツールの対応 |
|---|---|---|---|
| A. 数値/構造化データで立体化 | 測点・オフセット・標高 | ◎ | **実装済み**（ロフトエンジン） |
| B. DWG/DXFから半自動 | レイヤ分けされたCAD | ○ | **実装済み**（図面ビュー＋平面トレース押し出し＋横断ジオリファレンス） |
| C. PDF/紙から全自動 | スキャン画像 | △ | 非対応（検討根拠には精度が出ず危険なため意図的に見送り） |

横断図をDWG/DXFから完全自動で3D化しないのは、横断が「図面空間の任意位置」に描かれ、
どの測点・どの標高基準かを機械が正しく読むのが不安定なためです（誤読は検討を誤らせます）。
そのため「平面で断面線2点＋横断でEL基準2点を指定してジオリファレンス」する半自動方式を採用しています。

## アーキテクチャ

```
src/
  main.js              エントリ。UI配線・図面/立坑/地盤/統合ビューの状態管理・レイキャスト操作
  style.css            パネル/HUDのスタイル
  core/
    alignment.js       線形モデル（追加距離→位置・接線・法線、オフセット位置）
    corridor.js        横断面を線形に沿ってロフト→BufferGeometry（デモ現場用）
    cad.js             DWG(LibreDWG/WASM)・DXF(dxf-parser)読込→レイヤ別ポリライン＋テキスト正規化
    georef.js          横断ジオリファレンス（EL基準の1次写像・断面のf再サンプル）
  viewer/
    coords.js          測量座標(東,北,標高)→three.js座標 の一元変換
    scene.js           レンダラ/カメラ/OrbitControls/照明/グリッド
    objects.js         デモ現場の3Dオブジェクト生成（サーフェス/横断線/試験掘り/構造物/仮設）
    drawing.js         DWG/DXF図面のレイヤ別2D表示（図面ビュー）
    extrude.js         立坑ソリッド押し出し・試験掘り柱状図・構造物ボックス
    ground.js          ジオリファレンス断面の3D折れ線＋現況地盤サーフェスのロフト
  data/
    demo.js            デモ現場のサンプルデータ
  scripts/
    copy-wasm.mjs      LibreDWGのwasmを public/ へ配置
```

座標系は `viewer/coords.js` の `toWorld(東, 北, 標高)`＝(東=X, 標高=Y, 北=-Z) に一元化。
立坑・試験掘り・構造物・地盤サーフェスすべて同じ規約で配置され、統合ビューで正しく重なります。

## データモデル

```js
// 線形: 平面座標の点列
alignmentPoints = [{ x, y }, ...]            // x=東, y=北 (m)

// 横断: 測点ごとの現況/計画ライン
sections = [{ station, profile: [{ offset, z }] }]
//   station=追加距離(m), offset=中心からの水平距離(m,左が正), z=標高(m)

// 試験掘り
pits = [{ id, station, offset, layers: [{ name, thickness, color }] }]

// 構造物 / 仮設
structures = [{ id, type:'box', station, offset, width, run, height, sink, color, label }]
temp       = [{ id, type:'excavation', station, offset, width, run, depth, color, label }]
```

## 今後の実装候補（優先度順）

- ✅ **DWG/DXF読み込み（図面ビュー）** — 実装済み
- ✅ **平面図トレース→押し出しで立坑3D起こし** — 実装済み（底面積・掘削体積算出つき）
- ✅ **試験掘り柱状図の3D配置** — 実装済み（土層色分け・深さ比較）
- ✅ **構造物（MH・カルバート等）の配置と当たり確認** — 実装済み（半透明ボックスで干渉確認）
- ✅ **横断図の半自動ジオリファレンス→現況地盤サーフェス** — 実装済み（EL2点＋断面線でスケール不要）
- ✅ **統合ビュー（地盤・立坑・試験掘り・構造物の重ね合わせ）** — 実装済み（同一実標高系・GL基準調整）
1. **段付き掘削・法面（オープンカット）対応**（深さ方向に複数段・勾配を指定）
2. **試験掘りのCSV/座標一括インポート**、モデルの保存/読み込み（localStorage/JSON）
3. **地盤サーフェスに切土/盛土の色分け**（掘削と地盤の差分表示）
4. **glTF/OBJエクスポート**（他ソフトへの受け渡し）
5. **距離・面積・体積の計測ツール**

## 技術スタック

- Vite 5 + Vanilla JS（ESモジュール）
- three.js 0.160（OrbitControls）
- @mlightcad/libredwg-web 0.7（DWG読み込み・WASM）
- dxf-parser 1.1（DXF読み込み）

> DWGの WASM(約9.4MB) は `npm install` 時に `public/wasm/` へ自動コピーされます（リポジトリには含めません）。

## 注意

- これは検討・可視化のためのプロトタイプです。設計成果物・出来形の確定には使わないでください。
- DXFは平面図の線形（ポリライン）読み込みに対応。DWGは未対応（DXFへ変換してください）。
